A task is the activity that is done to accomplish something
A task is atomic, Wide range or high level task definition probably cause inconsistency
A task period shall be done in one day, if the user failed to finish that task in one day due to urgent events or failure in correct task definition, then this task should be seprated into multiple related task. (system doesn't support subtasks yet)
A task shall not be periodic for a long time, it has to be distinct. (constant tasks behavior is not studied on the system yet, but for analysis a constant parameter will not positively affect data quality not analysis)
A task shall not be a big goal or a milestone, Milestones and big goals are defined by the user out of the system
